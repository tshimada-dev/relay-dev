## 振り返り（Retrospective）

- **うまくいった点**: Phase 3/4 で詳細な設計とタスク分解を行ったことで、実装フェーズでの手戻りが最小限に抑えられた。特に Path Traversal や DoS 対策を初期段階から設計に組み込んだことが、スムーズなセキュリティゲート通過につながった。
- **改善すべき点**: プロンプトの長さ制限という、外部 API 入力における基本的なバリデーションを見落としていた。Phase 5-2 で繰り返し指摘を受けることになったため、今後は設計テンプレートのチェックリストに「外部入力の最大長制限」を含めるべきである。
- **次回への申し送り**: 外部 SDK を使用するプラグインでは、SDK 自体のタイムアウト設定や非推奨警告（今回発生した `google-genai` への移行推奨など）を早期に調査し、設計に反映させることで将来のメンテナンスコストを低減できる。

## 技術的負債

**今回の実装で発生した妥協点・将来対応すべき項目**

| 項目 | 内容 | 理由 | 優先度 | 対応期限目安 |
|------|------|------|--------|-------------|
| プロンプト長制限 | `prompt` 引数に対する最大文字数制限（例: 10,000文字）の実装 | メモリ保護および DoS 対策の強化。現状は API 側の制約に依存。 | 低 | 次回メンテナンス時 |
| SDK タイムアウト設定 | `generate_content` 等の呼び出しに対する明示的なタイムアウト指定 | ネットワーク遅延発生時のスレッド占有防止。 | 中 | リリース後1ヶ月以内 |
| SDK 移行 | `google-generativeai` から `google-genai` への移行 | 現行 SDK の Deprecated 警告（SDK内部）への対応。 | 低 | 半年以内 |
