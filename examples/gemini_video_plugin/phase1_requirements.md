# Phase1 要件抽出: gemini_video プラグイン追加

## 思考プロセス

1. **事実と推測の分離**:
    - **事実**: 
        - Gemini API を使用した動画解析プラグイン `gemini_video` を新規追加する。
        - MCP ツール `analyze_video_gemini` を提供し、動画パスとプロンプトを入力とする。
        - 実装は `plugins/gemini_video/plugin.py` の `GeminiVideoPlugin` クラス。
        - `BasePlugin` 継承、`loguru` によるログ出力、同期実装（`asyncio.to_thread` 許容）。
        - 既存プラグインの変更は禁止。
    - **不明点（推測）**:
        - 使用する Gemini モデル（`gemini-1.5-flash` または `gemini-1.5-pro` 等）【要確認】。
        - サポートする動画ファイル形式（mp4, mov 等の一般的形式と推測）【推測】。
        - 1ファイルあたりの最大サイズ制限（Gemini API の制限に従うと推測）【推測】。
        - 解析結果の言語（プロンプトに依存するが、システムとしては日本語を主眼とすると推測）【推測】。

2. **目的・KPIのドラフト**:
    - 目的: 動画ファイルを Gemini API で高度に解析（異常検知・要約等）できるツールを MCP 経由で提供し、IoT 監視の自動化・高度化を実現する。
    - KPI: 
        - 正常系テスト通過率 100%
        - API呼び出しから結果返却までのエラー率 1% 未満（ネットワーク・API制限起因を除く）
        - 既存システムへの影響（リグレッション）ゼロ

3. **スコープ定義**:
    - **What**: `gemini_video` プラグインの実装、ツール定義、API 連携、設定バリデーション、単体テスト。
    - **Exclude**: 動画のリアルタイムストリーミング解析、動画編集機能、他プラグインのロジック変更。

---

### 目的

Gemini API を活用して、ローカルに保存された動画ファイルの内容（シーン、異常、要約など）を自然言語で解析できる MCP ツール `analyze_video_gemini` を追加し、システムの解析機能を拡張する。

### 成功条件（KPI/評価基準）

| 条件 | 計測方法 | 目標値 |
|------|----------|--------|
| 解析ツールの正常動作 | `test/test_gemini_video.py` の実行 | 全テストケース合格 |
| MCP 通信の整合性 | `execute_tool` の戻り値が `TextContent` リストであること | 仕様準拠 |
| エラーハンドリング | 無効なパスや API キー欠如時の適切なエラー出力 | loguru でのエラー記録 |
| 既存機能の維持 | 既存の `test/test_mcp.py` の実行 | デグレなし |

### 制約条件

- **技術**: 
    - `BasePlugin` クラスの継承必須。
    - Python 3.10+、Pydantic v2、loguru 使用。
    - print/stdout の使用禁止（loguru のみ）。
    - 非同期 MCP 環境下での同期実装（ブロッキング呼び出しは `asyncio.to_thread` ラップ）。
- **リソース**: Gemini API キーが必要。
- **納期**: 特になし。
- **その他**: 既存プラグイン（`video_analysis` 等）への変更禁止。

### 前提条件

- ユーザーは有効な Google Cloud Gemini API キーを所有している。
- 解析対象の動画ファイルは MCP サーバーがアクセス可能なローカルパスに存在する。
- `google-generativeai` ライブラリがインストール可能である【仮定】。

### 入力情報の欠損リスト

- 推奨される Gemini モデル ID（`gemini-1.5-flash` 等）
- API 呼び出し時のタイムアウト秒数
- 動画ファイルサイズの最大許容値（システム側での制限有無）
- `requirements.txt` への追加パッケージ（`google-generativeai` 等）の要否

### 実行スコープ

- **含める作業**:
    - `plugins/gemini_video/` ディレクトリおよび `plugin.py` の作成。
    - `analyze_video_gemini` ツールの定義と実装。
    - `config/config.yaml` 向けの Pydantic 設定モデル定義。
    - 外部 API 呼び出しのモックを含む単体テストの作成。
- **除外する作業**:
    - `video_analysis` プラグインなど既存コードの修正。
    - 動画ファイルのアップロード/管理機能（既存パスの利用のみ）。
    - 永続的な解析結果の DB 保存（本タスクのスコープ外）。

### リスク要因

| リスク | 影響度（高/中/低） | 発生確率（高/中/低） | 対策案 |
|--------|---------------------|----------------------|--------|
| API クォータ制限/コスト増加 | 中 | 中 | エラーハンドリングの実装、モデル選択の柔軟性確保 |
| 大容量動画によるメモリ/タイムアウト | 高 | 中 | タイムアウト設定の適切化、ファイルサイズ制限の検討 |
| API 仕様変更による互換性喪失 | 中 | 低 | ライブラリバージョンの固定、抽象化層の検討 |

### 次フェーズ用質問リスト

- 使用するデフォルトの Gemini モデルは何にしますか？ (例: `gemini-1.5-flash`, `gemini-1.5-pro`)
- `requirements.txt` に `google-generativeai` を追加してもよろしいですか？
- API キーは `config.yaml` の `plugins.gemini_video.api_key` に配置する形でよいですか？
- 動画解析のタイムアウト（API 待ち時間）の希望値はありますか？（例: 60秒）

## 要約（200字以内）
Gemini API を利用し動画解析を行う `gemini_video` プラグインを新規実装する。`analyze_video_gemini` ツールを提供し、自然言語によるシーン解析・異常検知を実現。既存プラグインは変更せず、`BasePlugin` 準拠の同期実装とする。最大のリスクは API 制限と大容量動画によるタイムアウト。次フェーズではモデル ID や追加パッケージの要否、設定ファイルの構造を確認する。
