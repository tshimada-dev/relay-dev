# Gemini Video Plugin 実装例

このディレクトリは、**relay-dev** を実際に稼働させて自動生成された成果物のサンプルです。
AIエージェント（Gemini 2.5 Pro）が、要件定義（Phase 1）からリリース判定（Phase 8）までのプロセスを自律的に完走した際の実生の出力内容が含まれています。

## プロジェクト概要

「MCP（Model Context Protocol）を使用して、任意の動画をGemini APIで解析・要約する動画解析プラグインを追加する」というタスクをAIエージェントに依頼した結果です。

## 注目ポイント

relay-dev は単なるコード生成ツールではなく、**SDLC（ソフトウェア開発ライフサイクル）を模倣**します。以下のファイルから、エージェント同士がどのようにレビューし合い、タスクをブレイクダウンしていくかを確認できます。

1. **要件定義と設計**
   - `phase1_requirements.md`: ユーザーの曖昧な要求から具体的なシステム要件を抽出しています。
   - `phase3_design.md`: 要件を元に、モジュール構成やAPI仕様などのシステム設計へ落とし込んでいます。

2. **タスク分解**
   - `phase4_task_breakdown.md`: 設計内容を実装可能なタスク粒度（T-01, T-02など）に分解し、タスクの依存関係を定義しています。

3. **個別のタスク実装とテスト (`tasks/` ディレクトリ)**
   - `tasks/` フォルダ配下に、タスクごとの変更履歴（`phase5_implementation.md`）や、完了ごとのチェック結果（`phase5-1_completion_check.md`）、脆弱性検査結果、テスト実行ログが詳細に記録されています。

4. **品質ゲートとレビュー（PRレビュー）**
   - `phase7_pr_review.md`: すべてのタスクが完了した後、最終的なコードレビューが行われ、マージ可否が判定されています。

---
このサンプルを見ることで、relay-dev フレームワークがどのような粒度でドキュメントを取り回し、エージェント間の文脈を維持しているかをイメージすることができます。
