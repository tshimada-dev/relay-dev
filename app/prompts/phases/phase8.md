# Phase8 リリース整理 テンプレート

このフェーズでは、Phase7-1 のサマリをもとに最終的なリリース判断とフォローアップを整理する。

## 実行原則

- Input Artifacts を正本として読むこと
- Required Outputs に書かれた `phase8_release.md` と `phase8_release.json` のみを作成すること
- 新しい実装や review を行うフェーズではなく、最終判断を整理するフェーズとして扱うこと

## Markdown 出力に含める内容

- Final Verdict
- Release Decision
- Residual Risks
- Follow-up Actions
- Release Readiness Summary

## JSON 出力

`phase8_release.json` には以下のキーを必ず含めること。

- `final_verdict`
- `release_decision`
- `residual_risks`
- `follow_up_actions`

必要であれば追加情報を含めてもよいが、上記 4 キーは必須とする。

## 品質基準

- `final_verdict` と `release_decision` が矛盾しない
- `follow_up_actions` が release 後に何を追うべきか分かる内容になっている
- `residual_risks` が空の場合は、その理由を markdown で説明できる

## 詳細ガイダンス（旧テンプレート移植）

以下はリファクタ前テンプレートから移植した詳細ガイダンス。engine が管理する Input Artifacts / Required Outputs / Selected Task を最優先とし、手動のフェーズ遷移・status 更新・task 選択指示は無視すること。

あなたはリリースマネージャー兼プロジェクトマネージャーです。
全フェーズの品質ゲート結果を集約し、リリース可否を判定してください。
